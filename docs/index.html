Why waifu-sprites?

Modern AI agents are smart, but their interfaces are boring command-line windows or bloated 500MB+ Web UIs. waifu-sprites fixes this by decoupling the Brain from the Face.

⚡

Minimal Resource Usage

Browser does the rendering work. No GPU, no Electron, no bloat. Runs on anything with a browser.

🎨

24 Animated Sprites

12 action poses + 12 emotion states, each with MP4 video variants. AI-generated, human-curated.

🔊

Kokoro TTS

Built-in text-to-speech with chunk queuing, skip controls, and adjustable playback speed.

🔧

Agent Hooks

Python hooks for hermes-agent — emotion detection, visual state changes, TTS chunking, all automatic.

🏠

Moddable

Edit HTML/CSS/JS, refresh browser, done. Add your own character sets by dropping files in assets/.

📺

MP4 Native

Drop .mp4 files in videos/ — auto-discovered, autoplay, loop. No transcoding, no build step.

Architecture

Decoupled design — the agent brain (WSL2/Python) talks to the display face (Node.js/browser) over HTTP.

┌─────────────────────────────────────────────────────┐ │ waifu-sprites (Windows native) │ │ │ │ server.js (:8000) tts_server.py (:8001) │ │ Sprite display server Kokoro TTS server │ │ ├─ POST /state ├─ POST /tts │ │ ├─ GET /current_state ├─ POST /clear │ │ ├─ GET /tts/status ────────┤ GET /tts/status │ │ ├─ POST /tts/skip ─────────┤ POST /tts/skip │ │ └─ POST /tts/clear ────────┘ │ │ │ │ index.html (browser) │ │ ├─ Sprite display + manual browse │ │ └─ TTS controls (⏮ ⏹ ⏭ chunk display) │ └─────────────────────────────────────────────────────┘ ▲ ▲ │ HTTP :8000 │ file queue │ │ ┌────────┴────────────────────────┴────────┐ │ WSL2 / hermes-agent │ │ │ │ waifu_hook.py (symlink → src/) │ │ ├─ set_waifu_state() → :8000 │ │ ├─ on_agent_reply() → queue file │ │ └─ emotion detection, TTS chunking │ │ │ │ waifu.py (symlink → src/) │ │ └─ Monkey-patches HermesCLI │ └──────────────────────────────────────────┘

Getting Started

Three things to run: the sprite server, the TTS server, and the agent hooks.

🚀

1. Sprite Server

node server.js
Serves the browser UI on port 8000. Handles sprite state, MP4 discovery, and TTS proxying.

🔊

2. TTS Server

python tts_server.py
Kokoro TTS on port 8001. Generates speech audio, manages chunk queue and playback.

🤖

3. Agent Hooks

Symlink src/ into hermes-agent. The monkey patch in waifu.py auto-injects hooks on CLI startup.
